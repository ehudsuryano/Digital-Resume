Developing a generic Infrastructure software suite in C# for testing web pages using Selenium web driver

About the Framework

A robust, scalable, and maintainable software suite developed in C# for testing web applications using Selenium WebDriver. This framework leverages modern testing principles and tools to ensure high-quality automation testing for diverse web applications.

Key Features

  • Cross-Browser Compatibility: Works seamlessly on all major browsers (Chrome, Firefox, Edge, Safari).
  • Page Object Model (POM) Architecture: Simplifies test maintenance and improves readability.
  • Detailed Reporting: Generates interactive HTML reports using Extent Reports, including screenshots for failed steps.
  • Layered Architecture: Separates concerns into Test, Service, Page, and Utility layers for maintainability.
  • Scalability: Easily integrates with CI/CD pipelines and supports parallel test execution.

Technologies Used

  • Programming Language: C#
  • Automation Framework: Selenium WebDriver
  • Design Pattern: Page Object Model (POM)
  • Reporting Tool: Extent Reports
  • Test Execution: Supports parallel and sequential execution

How It Works

The framework initializes tests by dynamically configuring WebDriver based on test parameters. It uses POM classes for interacting with web elements and provides step-by-step logging during execution. At the end of a test run, detailed reports are generated, including screenshots and execution logs.
