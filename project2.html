Developing a generic Infrastructure software suite in Python for testing web pages using Selenium web driver

About the Framework

A powerful and scalable infrastructure software suite developed in Python for automating web application testing using Selenium WebDriver. This framework combines modern testing principles with versatile tools to ensure high-quality and maintainable test automation.

  • Source code --work in progress

Key Features

  • Cross-Browser Compatibility: Supports Chrome, Firefox, Edge, and Safari, including headless testing.
  • Page Object Model (POM): Simplifies test development with reusable, maintainable page-specific modules.
  • Comprehensive Reporting: Generates detailed HTML reports using pytest-html and interactive, visually rich reports using Allure Reports.
  • Layered Architecture: Organized into separate layers for:
    • Test cases
    • Page interactions
    • Business logic
    • Utilities (e.g., configuration, logging)
  • Scalability: Supports parallel execution and integrates seamlessly with CI/CD pipelines.

Technologies Used

  • Programming Language: Python
  • Automation Framework: Selenium WebDriver
  • Test Framework: Pytest
  • Reporting Tools: pytest-html, Allure Reports

How It Works

The framework initializes tests dynamically based on configurations, including browser type and execution mode (e.g., headless). It uses POM classes to perform UI interactions and organizes test logic across distinct layers. During test execution, it logs each step and generates detailed reports with actionable insights, including screenshots for failures.
